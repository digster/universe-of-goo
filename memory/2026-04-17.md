# 2026-04-17 ‚Äî Universe of Goo: initial implementation

Built the entire project from scratch in one session. Greenfield directory
at `/Users/ishan/lab/experiments/universe-of-goo`.

## What ships

### Three browser entry points
- `index.html` ‚Äî playable game, loads `level-01` by default.
- `sandbox.html` ‚Äî free-form physics playground.
- `docs/index.html` + `docs/pages/01-07.html` ‚Äî seven-page documentation
  site with live canvas demos.

### Physics core (`src/physics/`)
- `vec2.js` ‚Äî math helpers including `closestPointOnSegment` and
  `segmentsIntersect`.
- `particle.js` ‚Äî `makeParticle`, `pin`, `unpin`, `warpTo`,
  `applyForce`. Fields: `x, y, oldX, oldY, fx, fy, invMass, radius, userData`.
- `integrator.js` ‚Äî `verletStep(p, dt, gx, gy, damping)`; four lines.
- `constraint.js` ‚Äî `makeConstraint`, `solveConstraint` with
  invMass-weighted split correction, `stressRatio`, `maxStretch` break.
- `collision.js` ‚Äî `resolveSegment` with tangential damping.
- `world.js` ‚Äî orchestrates `step(dt) = integrate ‚Üí N√ó(relax+collide+bounds) ‚Üí prune`.

### Game layer (`src/game/`)
- `gootypes.js` ‚Äî registry of 4 types: `basic`, `balloon` (lift 420),
  `sticky` (3 strands, high stiffness), `fixed` (pinned, Infinity stretch).
- `gooball.js` ‚Äî wraps a particle with type, color, gaze, strand list;
  `applyPreStepBehaviour` adds balloon lift with half-force for free balloons.
- `strandRules.js` ‚Äî `pickAttachmentCandidates(pos, balls, segments, {K, R, LOS})`
  with distance sort, line-of-sight segment intersection check, maxStrands cap.
- `level.js` ‚Äî JSON schema validator + `buildLevel(spec) ‚Üí {world, balls, goal}`;
  `fetchLevel(name)` uses plain relative URL (not import.meta.url ‚Äî that broke).
- `goal.js` ‚Äî trigger-zone goal pipe; consumes free balls within radius.
- `sandboxController.js` ‚Äî spawn, link, delete, pin, clear, save/restore with
  localStorage key `universe-of-goo.sandbox.v1`.

### Rendering / input
- `renderer.js`, `drawPrimitives.js`, `debugOverlay.js` ‚Äî canvas 2D draw
  with sagging bezier strands, gaze-tracking eyes, stress-colored debug.
- `pointer.js` ‚Äî unified mouse/touch, maps CSS to canvas-internal coords.

### Tests
- **Vitest**: 36 unit tests across particle, constraint, collision,
  strandRules, level. All pass.
- **Playwright**: 19 smoke tests across game, sandbox, docs. All pass.
  Runs against the built `dist/` via `vite preview` on port 4173.

## Key decisions
- **Multi-page docs** over single-scroll ‚Äî deep-linkable, each page loads
  only what it needs.
- **4 goo types** (basic, balloon, sticky, fixed) ‚Äî minimum to showcase
  per-type behaviour without ballooning scope.
- **Hand-rolled physics** (no Matter.js) ‚Äî the codebase exists to teach
  the physics; a library hides the interesting part.
- **`base: './'`** in vite config ‚Äî built docs open from Finder with no
  server.

## Gotchas worth remembering
See `LEARNINGS.md` for the full list. Highlights:
- ES modules don't load over `file://`; dev needs `npm run dev`.
- Canvas y is down; collision tests often flip direction if you don't
  draw the scene on paper first.
- Both-pinned constraints would divide by zero in the solver; the solver
  early-returns to handle it. Do not remove that branch.
- `.overlay[hidden] { display: none; }` is needed because `display: grid`
  on `.overlay` otherwise wins by source order.
- Playwright `h1` locator picks the sidebar heading; scope to `.docs-main h1`.
