02 · Two-particle spring

A single distance constraint between two particles is the building block of every rope, cloth, tower and goo strand in the game. Instead of integrating spring forces, we project positions directly — a trick popularised by Thomas Jakobsen (Hitman's ragdolls, 2001).

Position-based, not force-based

A classical Hookean spring adds a force F = k · (len - restLen) to each endpoint each frame. That works but needs very small timesteps at high stiffness, otherwise the spring overshoots and explodes.

The Verlet world lets us do something simpler: measure the current distance, compute how far each endpoint needs to move to restore the rest length, and just move them. Velocity takes care of itself because it's encoded in the old position.

The relaxation step

This is the complete solver for a single stick (src/physics/constraint.js): 

d     = b.pos - a.pos
len   = |d|
if len > restLength · maxStretch: broken = true; return
diff  = (len - restLength) / len
wA    = a.invMass / (a.invMass + b.invMass)
wB    = 1 - wA
a.pos += d · (stiffness · diff · wA)
b.pos -= d · (stiffness · diff · wB)

One pass usually isn't enough: fixing constraint A can violate constraint B. The fix is to loop — run the pass N times per step. More iterations ⇒ stiffer system. The slider below lets you feel that trade-off.

Splitting the correction by inverse mass means a pinned particle (invMass = 0) absorbs zero correction, and its partner absorbs the full amount. No special case needed in the solver.

Live demo

Two balls, one constraint. Drag either ball to perturb. Crank iterations low and you'll see the constraint visibly stretch before recovering over several frames. Set stiffness to 0.2 and it'll sag like a rubber band.

Code map

  • makeConstraint(a, b, {restLength, stiffness, maxStretch}) — factory; defaults restLength to current distance.
  • solveConstraint(c) — the math block above.
  • stressRatio(c) — returns len / restLength, used by the tower demo to tint under-tension edges.
