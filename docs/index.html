Universe of Goo

A tiny, teaching-first browser game inspired by World of Goo. The physics engine is hand-rolled — Verlet integration, Jakobsen-style constraint relaxation, circle-vs-segment collisions — so the documentation can walk through the real modules that render every frame of the game.

How to read these docs

Each page introduces one concept, then drops you into an interactive canvas powered by the same source files the game imports. The demos live in docs/demos/*.js and import straight from src/physics/* — no simplified copies, no pedagogical forks. If a slider changes behaviour in a demo, it would change behaviour in the game too.

Read top-to-bottom for a full pass (each page pager-links to the next), or jump by concept from the sidebar.

Concepts

01

Particle & Verlet

Why storing a previous position is enough to represent velocity, and how one line of math integrates gravity without ever computing a derivative.

 02

Two-particle spring

One distance constraint between two particles. Tune stiffness, rest length, and iteration count to see how iterative relaxation converges.

 03

Chain / rope

N linked particles. Drag the tail to feel tension propagate; crank maxStretch down to snap the chain.

 04

Cloth grid

Structural & shear constraints over a lattice. Pinned corners anchor the cloth; drag free particles to deform it.

 05

Tower stability

A triangulated stack under gravity. Constraints are tinted by stress — watch load paths light up under load.

 06

Full goo ball

Drop balls near a seed tower; watch the K-nearest with line-of-sight rule decide which strands form.

 07

Balloon lift

Per-particle force accumulators. One ball lifts an entire chain when the attached weight is under budget.

Running the docs offline

These docs are set up so the built version works with no server. After npm run build, the dist/docs/ tree uses relative asset paths (base: './' in vite.config.js) — so double-clicking dist/docs/index.html in Finder loads the whole site from the filesystem.

In dev, run npm run dev — the browser refuses to resolve ES module imports over file://, so authoring requires the Vite server.
